# Library Project

## Описание

Library Project — это веб-приложение для управления библиотечными ресурсами. Приложение предоставляет функциональность для управления книгами, пользователями и процессами выдачи книг. Оно предназначено для упрощения администрирования библиотек, предоставляя удобный интерфейс для добавления, редактирования и удаления книг, а также для управления пользовательскими записями и записями о выдаче книг.

## Основные функции

- **Управление книгами**: Добавляйте новые книги в библиотеку, редактируйте информацию о книгах и удаляйте их из системы.
- **Управление пользователями**: Регистрация новых пользователей, редактирование их данных и удаление учетных записей.
- **Процессы выдачи книг**: Отслеживание текущих выданных книг, управление сроками возврата и проверка наличия книг.
- **Административная панель**: Управление всеми аспектами приложения через встроенную панель администратора Django.

## Технологии

- **Django**: Высокоуровневый фреймворк для веб-разработки на Python, который упрощает создание сложных веб-приложений.
- **PostgreSQL**: Мощная реляционная база данных, используемая для хранения информации о книгах, пользователях и процессах выдачи книг.
- **Docker**: Платформа для контейнеризации приложений, обеспечивающая изоляцию и упрощение развертывания приложений.
- **Docker Compose**: Инструмент для управления многоконтейнерными Docker приложениями, позволяющий легко настраивать и запускать приложения, состоящие из нескольких сервисов.

## Цели проекта

Library Project предназначен для автоматизации процессов управления библиотечными ресурсами. Он позволяет библиотекарям и администраторам:

- Упростить администрирование библиотеки, используя графический интерфейс.
- Эффективно управлять большими объемами данных о книгах и пользователях.
- Обеспечить удобный доступ к функциональности через веб-интерфейс.
- Использовать современные технологии для надежности и масштабируемости приложения.

# URL-ы и их назначение

В этом разделе описаны URL-ы, доступные в приложении Library Project, и их назначение.

## Управление книгами

- **Список книг**
  - **URL**: `/books/`
  - **Метод**: GET
  - **Описание**: Показывает список всех книг в библиотеке.

- **Просмотр книги**
  - **URL**: `/books/<id>/`
  - **Метод**: GET
  - **Описание**: Отображает подробную информацию о книге с указанным `id`.

- **Добавление книги**
  - **URL**: `/books/`
  - **Метод**: POST
  - **Описание**: Страница для добавления новой книги. GET-запрос отображает форму, POST-запрос отправляет данные книги на сервер.

- **Редактирование книги**
  - **URL**: `/books/<id>/`
  - **Метод**: PUTCH
  - **Описание**: Страница для редактирования информации о книге с указанным `id`. GET-запрос отображает форму с текущими данными, POST-запрос отправляет обновленные данные на сервер.

- **Удаление книги**
  - **URL**: `/books/<id>/`
  - **Метод**: DELETE
  - **Описание**: Удаляет книгу с указанным `id` из библиотеки. Запрос требует подтверждения удаления.

## Управление пользователями

- **Список пользователей**
  - **URL**: `/users/`
  - **Метод**: GET
  - **Описание**: Показывает список всех зарегистрированных пользователей библиотеки.

- **Просмотр пользователя**
  - **URL**: `/users/<id>/`
  - **Метод**: GET
  - **Описание**: Отображает подробную информацию о пользователе с указанным `id`.

- **Добавление пользователя**
  - **URL**: `/users/register/`
  - **Метод**: GET, POST
  - **Описание**: Страница для добавления нового пользователя. GET-запрос отображает форму, POST-запрос отправляет данные пользователя на сервер.

- **Редактирование пользователя**
  - **URL**: `/users/<int:pk>/`
  - **Метод**: PUTCH
  - **Описание**: Страница для редактирования информации о пользователе с указанным `id`. GET-запрос отображает форму с текущими данными, POST-запрос отправляет обновленные данные на сервер.

## Процессы выдачи книг

- **Список выданных книг**
  - **URL**: `/books/logs/`
  - **Метод**: GET
  - **Описание**: Показывает список всех выданных книг и информацию о том, кто и когда их взял.

- **Выдача книги**
  - **URL**: `/books/get_books`
  - **Метод**: POST
  - **Описание**: Страница для записи новой выдачи книги. Для передачи необходимо передавать список из одной или нескольких книг. GET-запрос отображает форму для ввода данных, POST-запрос отправляет данные на сервер. 

- **Возврат книги**
  - **URL**: `/books/return_books/<int:pk>`
  - **Метод**: PUTCH
  - **Описание**: Обрабатывает возврат книги с указанным `id` Журнала (LogService). Запрос обновляет статус книги и возвращает её в библиотеку.

## Административная панель

- **URL**: `/admin/`
- **Метод**: GET
- **Описание**: Панель администратора Django для управления всеми аспектами приложения, включая книги, пользователей и процессы выдачи.

## Дополнительные URL

- **Авторизация пользователя**
  - **URL**: `/login/`
  - **Метод**: GET, POST
  - **Описание**: Страница для входа в систему. GET-запрос отображает форму авторизации, POST-запрос отправляет учетные данные для проверки.

- **Регистрация пользователя**
  - **URL**: `/register/`
  - **Метод**: GET, POST
  - **Описание**: Страница для регистрации нового пользователя. GET-запрос отображает форму регистрации, POST-запрос отправляет данные на сервер.

- **Выход из системы**
  - **URL**: `/logout/`
  - **Метод**: POST
  - **Описание**: Выполняет выход пользователя из системы.

## Запуск проекта

1. Постройте и запустите контейнеры Docker:

   ```bash
   docker-compose up -d --build

2. Выполните миграции базы данных:

   ```bash
   docker-compose exec habit python manage.py migrate

3. Создайте суперпользователя (если необходимо):

   ```bash
   docker-compose exec habit python manage.py createsuperuser

4. Откройте браузер и перейдите по адресу http://localhost:8000/library/ для доступа к приложению.
